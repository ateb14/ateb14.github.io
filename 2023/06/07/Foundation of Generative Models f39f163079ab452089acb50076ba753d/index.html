Foundation of Generative Models

Posted on 2023-06-07 In Generative Models

Foundation of Generative Models

Definition

Given observational data sampled from the true distribution \(p(x)\), denoted as \(x \sim p(x)\), training (parameter estimation) can be conducted to obtain a generative model \(p_{\theta}(x)\) controlled by \(\theta\) that approximates the true distribution.

Difficulties

Generally speaking, we cannot directly sample \(x\) from \(p_{\theta}(x)\), but it is easy to sample from a uniform distribution or a Gaussian distribution (using re-parameterization trick) to obtain data. How to use a simple distribution or other methods for sampling is also a consideration in the modeling process.

Models

Latent Variable Models

\(p(x) = \int p(x|z)p(z)dz\), where \(z\) follows the distribution \(p(z)\). \(z\) is called a latent variable, and \(p(z)\) is a prior probability distribution that can be arbitrarily specified. A model of this form is called a latent variable model.

For a single latent variable model, we can approximate \(p(x|z)\) by \(p_{\theta}(x|z)\). Once we have done so, it is easy to compute \(p_{\theta}(x) = \int p_{\theta}(x|z)p(z)dz\). By using the prior \(p(z)\), we can approximate the joint probability \(p(x,z)\), which is equivalent to approximating the likelihood \(p(x|z)\) since \(p(x,z) = p(x|z)p(z)\).

\[ \begin{aligned}KL(p(x,z)||p_\theta(x,z)) &= \int\int p(x,z)log\frac{p(x,z)}{p_\theta(x,z)}dzdx\\ &= \mathbb{E}_{x\sim p(x)}[logp(x)] + \mathbb{E}_{x\sim p(x)}[\int p(z|x)log\frac{p(z|x)}{p_\theta(x,z)}dz] \\ &=C + L\mathbb{} \\ \\L &= KL(p(x,z)||p_\theta(x,z)) - C \\ &= \mathbb{E}_{x\sim p(x)}[\int p(z|x)log\frac{p(z|x)}{p_\theta(x,z)}dz]\end{aligned} \]

Minimizing \(KL(p(x,z)||p_\theta(x,z))\) is equivalent to minimizing \(L\), which is called the evidence lower bound (ELBO).

(Actually it’s the negative of ELBO: \(\mathbb{E}_{z\sim p(z|x)}[\int p(z|x)log\frac{p_\theta(x,z)}{p(z|x)}dz] \le \log p_\theta(x)\) )

\[ \begin{align} L &= \mathbb{E}_{x\sim p(x)}[\int p(z|x)log\frac{p(z|x)}{p_\theta(x,z)}dz] \\ &= \mathbb{E}_{x\sim p(x)}[\mathbb{E}_{z\sim p(z|x)}[-log p_\theta(x|z)] + KL(p(z|x)||p(z)) ] \end{align} \]

We refer to the neural network approximating \(p_\theta(x|z)\) as the decoder, and the neural network approximating \(p_\theta(z|x)\) as the encoder.

First, let's look at the first term of the optimization function L, where \(p_\theta(x|z)\) is the likelihood function. Therefore, minimizing \(-log p_\theta(x|z)\) is equivalent to maximizing the likelihood function. The second term aims to make the true posterior distribution \(p(z|x)\) as close as possible to the prior distribution \(p(z)\), so that the \(z\) sampled from the prior distribution can correspond to the data \(x\) and be fed into the decoder for better generation.

There are still some difficulties:

  • How to generate \(x\)? According to \(p_{\theta}(x) = \int p_{\theta}(x|z)p(z)dz\), we would need to sample an infinite number of \(z\) values to obtain \(p_\theta(x)\).
  • If we already have \(p_\theta(x)\), it remains unclear how to sample \(x\) from it.

Some possible solutions:

  • We can view \(p_\theta(x|z)\) as generating \(x\) from \(z\).
  • We can use \(\mu = \mathbb{E}_{x\sim p_\theta (x|z)}[x]\) to represent \(x\) generated by \(z\).

Then here comes some targets:

  • We can derive the maximum likelihood and expectation from the likelihood function \(p_\theta(x|z)\). This means that the formulation should be "nice".
  • The form of the posterior \(p(z|x)\) is sufficient for us to calculate its KL divergence with the prior \(p(z)\).

Autoencoder

The idea behind autoencoders is that the latent variable z and x can uniquely correspond to each other. The mapping from \(x\) to \(z\) belongs to feature dimensionality reduction, while the mapping from \(z\) to \(x\) belongs to feature reconstruction. This is equivalent to the mapping between high-dimensional space and low-dimensional manifold, which belongs to a single latent variable model, and the latent variable space is continuous and uniform.

  • Posterior: \(p(z|x)= \delta(z-C(x))\), where \(C(x)\) is the true encoder and \(\delta(x)\) is the Dirac delta function.
  • Prior: \(p(z) = \frac{1}{D}, 0\le z\le D\).
  • Likelihood: \(p_\theta (x|z) = N(G_\theta(z), \sigma^2I)\), where \(G_\theta(z)\) is the decoder.

Variational Autoencoder(VAE)

VAE supposes that each \(x\) should correspond to a distribution, and this distribution can be used to regenerate \(x\).

  • Prior: \(p(z) = N(0,I)\)
  • Posterior: \(p(z|x) = N(\mu,\sigma^2I), p_\theta(z|x) = N(C_\theta(x),\sigma^2I)\)
  • Likelihood: \(p_\theta (x|z) = N(G_\theta(z), \sigma^2I)\), where \(G_\theta(z)\) is the decoder.

An intuitive explanation from zhihu:

During training, since each \(x\) corresponds to a Gaussian distribution, there are many Gaussian distributions \(p_\theta(z|x)\) that aim to approximate \(p(z)\). As a result, during sampling, a \(z\) value may be “disputed” by multiple Gaussian distributions \(p_\theta(x|z)\), and it may end up resembling the one with the closest mean, creating a "chimera" that generates diversity while still achieving the desired outcome.

Diffusion

In contrast to the aforementioned single latent variable model, diffusion models involve several latent variables \(z_1,...,z_T\). The distribution of \(x\) is expressed using a Markov Chain approach.

\[ p(x) = \int p(x|z_1)p(z_1|z_2)...p(z_{T-1}|z_T)p(z_T)dz_{1:T} \]

Then the “Bayesian components” can be indicated as follows:

  • Prior: \(p(z_T) = N(0,I)\)
  • Posterior: \(p(z_T|x)= N(\mu, \sigma^2I), p(z_{t}|z_{t-1}) = N(\sqrt{\alpha_t}z_{t-1}, \beta_t)\) (In DDPM).
  • Likelihood: \(p(x|z_T) = p(x|z_1)p(z_1|z_2)...p(z_{T-1}|z_T)\)

We observe that the encoder(posterior) here is unlearnable(fixed) and as \(T\rightarrow \infty\), \(p(z_T,x)\rightarrow N(0,I) = p(z_T)\).

Flow-based Models

Flow models aim to establish a one-to-one correspondence between \(x\) and the latent variable \(z\), similar to the single latent variable model of AE. However, what sets flow models apart is that they also aim for this correspondence to be a bijection.

  • Posterior: \(p(z|x) = \delta(z-C(x))\)
  • Likelihood: \(p(x|z) = \delta(x-G(z)), G = C^{-1}\)
  • Prior: \(p(z) = N(0,I)\)

Summary for Latent Variable Models

  • The concept behind AE is that \(x\) and \(z\) have a one-to-one correspondence, while VAE assumes that \(x\) corresponds to a particular distribution. Both types of models use neural networks to directly fit their encoders and decoders.
  • Like VAE, diffusion also aims for \(x\) to correspond to a certain distribution, but it utilizes a chain-like structure for the encoder instead of directly simulating it with a neural network.
  • The Flow model is comparable to AE in its aim for a unique correspondence between \(x\) and \(z\), but it places greater emphasis on an inverse relationship between the encoder and decoder. As a result, its design is more rigorous than the encoder structure of Diffusion.

Energy-based Models

Consider directly modeling \(p(x)\) as:

\[ p_\theta(x) = \frac{q_\theta(x)}{Z(\theta)} \]

Here, \(Z(\theta) = \int q_\theta(x)dx\), which is known as the normalization constant.

Specifically, if \(q_\theta(x) = e^{-U_\theta(x)}\), then:

\[ p_\theta(x) = \frac{e^{-U_\theta(x)}}{Z(\theta)} \]

Here, \(U_\theta(x)\) is an unspecified function, also known as the energy function; and \(p_\theta(x)\) is called the energy distribution, also known as the Maxwell–Boltzmann Distribution.

Score Matching

Score function \(s_\theta(x)=\nabla_{x}\log p_\theta(x) = \nabla_{x}\log q_\theta(x)\)\(\nabla_{x}\log p_\theta(x) = \nabla_{x}\log q_\theta(x)\), which is irrelevant to \(Z(\theta)\). Estimating the gradient of \(\nabla_{x}\log p_\theta(x)\) can also provide insight into the data distribution, as they only differ by a scaling factor \(Z(\theta)\). This can be helpful in understanding the shape and trend of the data distribution \(p(x)\).

The optimization object can be defined using Fisher divergence:

\[ \begin{aligned} J(\theta) &=\frac{1}{2}\int p(x)||\nabla_{x}\log p_\theta(x) - \nabla_{x}\log p(x) ||^2 dx\\ &= \int p(x)[\frac{1}{2}||s_\theta(x)||^2 -s_\theta(x)^Ts_\theta(x) + \frac{1}{2}||s(x) ||^2]dx\end{aligned} \]

The third term in this equation can be neglected since it’s irrelevant to \(\theta\).p

For the \(i^{th}\) element of the second term here:

\[ \begin{aligned} -\int p(x)s_{\theta,i}(x)s_i(x)dx &= -\int s_{\theta,i}(x)\frac{\partial p(x)}{\partial(x_i)}dx \\ &= -\int[\int s_{\theta, i}(x)\frac{\partial p(x)}{\partial(x_i)}dx_i]dx_{1:n-\{i\}} \\ &= -\int[s_{\theta,i}(x)p(x)\bigg|_{x = -\infty}^{x=+\infty}-\int p(x)\frac{\partial s_{\theta,i}(x)}{\partial(x_i)} dx_i]dx_{1:n-\{i\}} \\ &= \int p(x)\frac{\partial s_{\theta,i}(x)}{\partial(x_i)}dx_{1:n} \end{aligned} \]

Then the second term is:

\[ \begin{aligned} -\int p(x)s_\theta(x)^Ts_\theta(x) dx&=\int p(x)tr(\nabla_\bold{x}s_\theta(x))dx_{1:n} \end{aligned} \]

Then the optimization object is :

\[ \begin{aligned} \int &p(\bold{x})[\frac{1}{2}||\nabla_x\log p_\theta (\bold{x})||^2 + tr(\nabla_x^2\log p_\theta (\bold{x}))]d\bold{x} \\&= \mathbb{E}_{x\sim p(x)}[\frac{1}{2}||\nabla_x\log p_\theta (\bold{x})||^2 + tr(\nabla_x^2\log p_\theta (\bold{x}))] \\&= \mathbb{E}_{x\sim p(x)}[\frac{1}{2}||s_\theta(x)||^2 + tr(\nabla_\bold{x}s_\theta (\bold{x}))].\end{aligned} \]

Here \(\nabla_x^2\log p_\theta (\bold{x})\) denotes the Hessian Matrix of \(\log p_\theta (\bold{x})\).
